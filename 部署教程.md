# 部署教程

本文档提供 **qq-farm-bot-ui** 的生产部署指引，覆盖 Docker 单机部署、源码部署、反向代理、升级回滚与故障排查。

## 1. 部署前提

- 推荐单机/单容器部署（当前架构按单写者设计）。
- 服务器开放端口（默认 `3000`，如走反代则仅内网开放）。
- 已安装 Docker + Docker Compose（推荐）或 Node.js 20+（源码部署）。

## 2. 推荐方式：Docker Compose（生产）

### 2.1 准备目录

```bash
git clone <your-repo-url> qq-farm-bot-ui
cd qq-farm-bot-ui
mkdir -p data
```

### 2.2 创建 `.env`

在仓库根目录创建 `.env`：

```env
ADMIN_PASSWORD=请改成高强度密码
ADMIN_JWT_SECRET=请改成至少16位随机字符串_建议32位以上
ADMIN_ALLOWED_ORIGINS=https://your-panel.example.com
ADMIN_TRUST_PROXY=loopback, linklocal, uniquelocal
ADMIN_BIND_TOKEN_IP=1
FARM_SINGLE_WRITER_STRICT=1
FARM_PERSIST_STRICT_CONFLICT=1
TZ=Asia/Shanghai
```

> 生产环境不要使用默认弱口令，不要将 `ADMIN_TRUST_PROXY` 设为 `true`。

### 2.3 启动

```bash
docker compose up -d --build
```

### 2.4 验证

```bash
docker compose ps
docker compose logs -f --tail=200
```

浏览器访问：

- 直连：`http://<server-ip>:3000`
- 反向代理：`https://your-panel.example.com`

## 3. 源码部署（可选）

```bash
corepack enable
npm install -g pnpm
pnpm install
pnpm build:web
ADMIN_PASSWORD='强密码' \
ADMIN_JWT_SECRET='至少16位随机串' \
ADMIN_ALLOWED_ORIGINS='https://your-panel.example.com' \
ADMIN_BIND_TOKEN_IP=1 \
pnpm dev:core
```

## 4. Nginx 反向代理示例

```nginx
server {
  listen 443 ssl http2;
  server_name your-panel.example.com;

  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
  }
}
```

并保持后端 `ADMIN_TRUST_PROXY=loopback, linklocal, uniquelocal`。

## 5. 升级与回滚

### 升级

```bash
git pull
docker compose up -d --build
```

### 回滚（示例）

```bash
git checkout <previous-tag-or-commit>
docker compose up -d --build
```

## 6. 常见问题

### 6.1 登录 401

- 检查 `ADMIN_PASSWORD` 是否正确。
- 检查 token 是否过期、浏览器本地 token 是否已失效。
- 若启用 `ADMIN_BIND_TOKEN_IP=1`，确认请求来源 IP 未发生异常变化。

### 6.2 跨域被拦截

- 确认 `ADMIN_ALLOWED_ORIGINS` 包含前端来源（协议+域名+端口）。

### 6.3 反向代理后限流异常

- 检查 `ADMIN_TRUST_PROXY` 是否设置为可信代理范围，不要设为 `true`。

### 6.4 数据写入冲突

- 保持 `FARM_SINGLE_WRITER_STRICT=1`、`FARM_PERSIST_STRICT_CONFLICT=1`。
- 避免多副本共享同一 `data/` 目录。

## 7. 生产上线检查清单

- [ ] 已设置强密码 `ADMIN_PASSWORD`
- [ ] 已设置高强度 `ADMIN_JWT_SECRET`（>=16）
- [ ] `ADMIN_ALLOWED_ORIGINS` 仅放行可信来源
- [ ] `ADMIN_TRUST_PROXY` 为可信代理策略（非 true）
- [ ] `ADMIN_BIND_TOKEN_IP=1`
- [ ] `FARM_SINGLE_WRITER_STRICT=1`
- [ ] `FARM_PERSIST_STRICT_CONFLICT=1`
- [ ] `data/` 目录已持久化并可备份
